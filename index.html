Experience Showcase

Powerboard for the Illini-SAT (Project Link)
This was my senior design project and was worked on by three senior students including myself. Our project goals and components were given by our sponsor, LASSI (Laboratory for Advanced Space Systems at Illinois). The main goal of our project was to build a new power board for their next generation model of CubeSats, a small cube shaped satellite, along with implementing I2C and CANBus protocols. The power board was requested to have the battery and solar powered panels be more energy efficient. I2C was requested in order to implement safety features for each components and additionally allow us to read and send each component’s telemetric status. Finally, CANBus protocols was used to communicate between our microcontroller and the control + data handling board. This communication would allow our sponsors to create and execute custom instructions for the power board. The approximate time we had in implementing our solution was roughly three months and we approximately followed an agile development cycle. Due to our project producing admirable results within a short amount of time, we additionally received an honorable award for our project. Overall, working on this project not only helped me learn the internal functionality of a power board and basic networking protocols, but also taught me about time management along with being able to read and write technical reports in an efficient manner.

BBQ Restaurant Analytic (Project Link)
This analysis focused on determining which variables made a larger impact in determining whether a restaurant was successful or not. The general notion of a successful restaurant can be summarized as good service, food, location, and reasonable pricing. Using the FourSquare API database, I gathered BBQ restaurants data scattered across the USA and created a dataset that consisted of the restaurant's name, location, rating, menu items, etc. Due to the many of the rows containing missing values, I evaluated that I cannot make a predictive model using the current dataset I have as this may not accurately represent the datasets actual distribution. Thus, I focused on making a cluster model that would help identify any patterns in the current the dataset. DBSCAN clustering turned out to be the one that was able to give a reasonable cluster and the resulting cluster showed number of rating reviews and pricing to stand out.

Android Facial Recognition (Github Link)
This was my digital signal processing lab final project and was worked on by three university students including myself. The project’s requirement was to implement a DSP algorithm from a written research paper, take that algorithm and create a meaningful product that users can use, and implement the algorithm on an android app. Our focused DSP algorithm was the eigenfaces and canny/sobel algorithm. The final intended implemented product utilizing these algorithms would be a security camera app. Our app would focus on being able to detect a person’s face and check if the person’s face was in the facial database. If the face was in the database, we would return the predicted name associated with the picture taken in real time. Additionally, our app would be able to take a picture of an object that a person was holding and determine whether the object was dangerous or not. The development cycle was roughly two months and was done on an iterative development cycle. Overall, the app managed to run relatively smooth and the identification portion took roughly from 2-4 seconds. This was to be expected as we have to apply histogram equalization and canny edge onto the captured frame image. The general lessons learned from doing this project came from implementing a research paper algorithm and constantly improving upon our implementation until we managed to hit all our desired milestones.
